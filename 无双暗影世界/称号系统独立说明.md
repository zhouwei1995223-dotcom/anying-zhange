# 称号系统独立说明

这份文档专门记录称号系统，不和饰品、时装、装备强化特效混在一起。

## 一、称号装备位

称号现在有独立装备位：

```xml
<title>称号道具ID</title>
```

示例：

```xml
<title>1713049900</title>
```

没有装备称号时：

```xml
<title></title>
```

称号不会占用耳环、戒指、项链、时装。

## 二、称号道具配置

配置文件：

```text
D:\无双暗影世界\无双暗影世界\server\config\data\jewels\jewels.txt
```

称号虽然暂时复用饰品数据结构，但必须设置：

```text
goodsType = 3
dType = 4
```

`dType=4` 的含义就是“称号”。

当前测试称号：

```text
1713049900 天下无双
```

## 三、称号属性

称号可以加属性，属性字段沿用饰品字段，例如：

- 生命上限
- 魔法上限
- 物理防御
- 魔法防御
- 命中
- 闪避

当前 `天下无双` 已经能正常参与角色属性计算。

## 四、称号特效

称号特效目前通过 `goodsLuck` 指定：

```text
goodsLuck = 特效编号
```

当前 `天下无双` 使用：

```text
goodsLuck = 18
```

客户端资源对应：

```text
D:\无双暗影世界\无双暗影世界\client\pak\effect.zip
```

里面的路径类似：

```text
image/effect/18.img
```

## 五、不要再使用 halo2.img 做称号

`halo2.img` 不是称号专用资源。

它原本属于装备强化等级特效：

- 强化等级不同
- 显示颜色不同
- 属于装备强化系统

称号不要继续改 `halo2.img`，否则会影响强化装备本来的特效。

后续新增称号时，建议使用独立编号资源：

```text
image/effect/30.img
image/effect/31.img
image/effect/32.img
```

然后在称号道具配置里写：

```text
goodsLuck = 30
```

这样称号效果就和装备强化 `halo2.img` 分开了。

## 六、新增称号流程

1. 复制 `jewels.txt` 里 `1713049900` 这一行。
2. 改成新的道具编号。
3. 改名称，例如 `仙尊临世`。
4. 保持 `dType=4`。
5. 设置称号属性。
6. 设置新的 `goodsLuck`，例如 `30`。
7. 把称号特效图片转成 `.img`。
8. 放进 `effect.zip` 的 `image/effect/30.img`。
9. 重启服务端和客户端。
10. 用 GM 指令添加新称号道具测试。

## 七、当前已修复逻辑

当前称号系统已经接入：

- 独立称号装备格
- 称号穿戴
- 称号脱下
- 称号属性加成
- 鼠标悬浮显示名称和属性
- 登录时显示称号特效
- 称号穿脱后刷新特效
- 时装穿脱后保留称号特效

## 八、常见问题

### 穿上称号没有特效

检查：

- `jewels.txt` 里的 `goodsLuck` 是否正确。
- `effect.zip` 里是否存在对应 `image/effect/编号.img`。
- 服务端是否重启。
- 客户端是否重启。

### 穿上时装后称号消失

这是外观刷新包没有带 `halo` 导致的。当前已经修复：时装穿脱后会重新发送完整外观包。

### 鼠标放称号上没有名称

这是客户端饰品类型表原本没有 `dType=4`。当前已经修复：客户端类型表增加了“称号”。
